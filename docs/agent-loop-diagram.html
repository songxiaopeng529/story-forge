📊 核心数据结构

概念类型说明
AgentLoopOptions { provider, tools, maxSteps?, maxDurationMs?, now? } 构造参数：模型提供者、工具注册表、步数/时间上限
AgentLoopRunInput { sessionId, turnId, responseMode?, messages, signal?, onEvent?, onCheckpoint? } 每次 run 的输入：会话/轮次 ID、响应模式、消息历史、回调
AgentLoopResult { messages, stopReason, steps } 运行结果：完整消息历史、停止原因、总步数
ResponseMode 'auto' | 'live' | 'smooth' 响应模式：自动降级 / 强制流式 / 非流式

🛑 停止原因一览

completed user-stopped time-limit step-limit repeated-tool-call consecutive-tool-failures unrecoverable-error

🔑 关键设计决策

决策 实现 目的
上下文窗口管理 trimMessagesToContext 从后往前保留完整对话轮次，最多 80% 窗口 防止 token 超限，同时保留最新上下文
流式降级 auto 模式下流式失败且无内容时降级到 smooth 提升可靠性，保证总能拿到响应
无限循环保护 默认 1000 步 / 2 小时上限；重复调用 3 次停止；连续失败 5 次停止 防止模型陷入死循环，控制成本
事件驱动架构 onEvent 推送 lifecycle / model / tool 事件 支持 UI 实时更新和调试
检查点机制 每轮 tool 执行后通过 onCheckpoint 保存快照 支持断点恢复和状态持久化

📡 事件类型

事件触发时机
runtime.startedAgentLoop 开始运行
runtime.completed正常结束（含 stopReason 和 steps）
runtime.error不可恢复的异常
model.request发送请求给模型前（inspect 模式）
message.delta模型返回内容增量（smooth 一次性 / live 逐片）
tool.call开始执行工具调用
tool.result工具执行完成
response.fallbackauto 模式流式降级到 smooth
